ghrunners

macOS 上 GitHub Actions self-hosted runner 的一次性、无守护进程 CLI——只读观测，外加受控的 launchd 控制。

{% include project-summary.html %}

是什么

ghrunners 会发现一台 Mac 上的每个 actions-runner， 交叉核对 launchctl 域状态、进程树、.runner JSON、worker 日志，以及（可选的）GitHub API。观测核心—— status、describe、doctor、 logs——把问题以类型化 findings 呈现；另有一个独立、 受控的 control 子命令，作用在同一份状态上。它是一个 私有的本地工具；它描述的快照名字写在下面的「证据」里。

要解决的问题

当一个 self-hosted runner 在 Mac 上悄无声息地挂掉，真相是 散开的：launchd 记着域和上次退出状态，进程树才知道 worker 是不是真活着，安装目录把配置锁在 0700 权限后面， 而 GitHub API 知道任务到底有没有落地。你 SSH 上去一点点拼， 而最顺手的修法——重启 runner——恰恰最危险：一条不小心的命令 就可能干掉正在跑的构建，或者把它 bootstrap 进错误的 launchd 域。

约束与关键决策

一次性观测，基于权威状态。不跑监控守护进程， ghrunners 是一次无状态的调用，它解析 launchctl print 的正文——不需要 sudo 也能读——所以它报出来的运行态和上次退出是权威的，即使是非 root 调用方，也不是从 ps 猜的。代价： 一次性意味着没有持续监控、也没有历史趋势——想要快照时你才跑它， 或者拿 cron 定时跑。

部分输出，而不是一遇错就停。runner 安装目录 在别的用户名下、权限 0700，所以一个严格的工具没有 sudo 就直接报错退出。这里换成优雅降级： PID - 和 REPO ? 不是错误，权限不足的 路径变成一条 Unreadable finding，而不是致命中断， 同时 state 列仍然从那份免 sudo 的 print 正文里保持 权威。代价：每一个空格子都是工具要设计并解释 的——是权限缺口，还是真的不存在——而且没有 sudo， 输出就是没那么全。

受控的控制，绑定在观测到的状态上。最顺手的 下一步——start / stop / restart——正是危险的那一步，所以 control（bootstrap / unload / restart）从观测核心 读到的同一份权威状态里解析目标，而不是自由发挥的 launchctl。--dry-run 只打印解析出来的 命令、并不执行；--domain 覆盖除了 DoubleLoaded 的 unload 之外，一律以用法错误拒绝（其余每个 verb 都有确定的 目标，覆盖只会把它搞乱）；而在某个 verb 可能干掉一个活着的 Runner.Worker 之前，必须显式传 --yes。 代价：要多带一堆前置条件和参数，而且工具会 故意拒绝一些原始 launchctl 照样会执行的命令。

类型化 findings 和稳定的退出码，而不是大段日志。 「这个 runner 健康吗」需要一个机器能读的答案，所以问题被收成 13 种带严重度的类型化 findings——doctor 只打印 可操作的 warn / error，每条带一行 fix:——而且当 status 或 doctor 看到 warning 或 error 时，进程以 1 退出，于是 ghrunners doctor >/dev/null || mail … 能当 cron 健康检查用。代价：每一条 finding 都是要定义、 触发、并指定严重度和修法的类型化契约——一份要维护的目录， 而不是自由文本。

证据

私有快照 local tag v0.4.0——没有可查看的公开源码。 支撑这些说法的，是正在运行的能力面：status、 describe、doctor、logs 和受控的 control 今天都能用，背后是一份 13 种 类型化 findings 的目录（从 LoadedNotRunning、 OrphanListener 到 DoubleLoaded、 Unreadable），带 cron 友好的退出码、JSON 输出， 以及每个控制动作上的 --dry-run 预览。这套能力是 分步落地的：v0.2 加了控制 verb，v0.3 让运行态从 print 正文里 变得权威，v0.4 加了 doctor 巡检和 OrphanListener。

下一步

候选范围，尚未敲定——更深的 GitHub API 信息补充，以及在真实 机器上随边角情况冒出来的新 findings。等仓库可以公开访问， 再加源码链接。

它不是什么

  • 不是 runner 安装或注销工具。
  • 不是 daemon 或持续监控器。
  • 不是 fleet 管理——控制是单 runner、受控的，不是批量编排。
  • 不支持 Linux 或 Windows。
  • 私有/本地——今天还不是公开源码项目。
