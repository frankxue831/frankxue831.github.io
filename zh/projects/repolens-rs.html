RepoLens

给编程 Agent 用的仓库记忆和上下文层，让新的会话 不必每次都从零开始理解项目。

{% include project-summary.html %}

是什么

RepoLens 把仓库扫描成结构化、不可变的「pack」， 通过 MCP server 暴露这套模型，再配上一张类型化、会随时间 衰减的记忆图。它不是替 Agent 写代码，而是让每个新会话不必 从零开始。这是一个私有预发布；它描述的快照名字写在下面的 「证据」里。

要解决的问题

编程 Agent 写代码不差，但会话一结束就丢掉项目上下文。每个 新会话都从冷启动开始：重新 grep 同样的文件、重新推导同样 的约定、重新犯几周前就被发现并回滚过的错误，因为没有任何 东西反驳，就把似是而非的猜测当成对的。最朴素的办法——更大 的上下文窗口、把更多文件塞进提示词——会把信号淹没在噪声里 （“lost in the middle”）。瓶颈不在智能，而在 持久、结构化的感知。

约束与关键决策

用结构化的类型化感知，而不是更大的提示词。 既然问题出在冷启动时的重新发现，而塞更多文件只会稀释信号， RepoLens 就把仓库扫描成一个不可变的类型化 pack——文件、 符号、依赖图、profile——并通过 26 个 MCP 工具（代码搜索、 图遍历、摘要、约定提取、记忆、wiki）暴露出来，而不是去加宽 上下文窗口。代价：pack 是一道构建并保持 更新的工序，而且因为 pack 不可变，它会一直旧到下一次扫描。

类型化、会衰减、有 grounding 的记忆，而不是一个 扁平存储。扁平的 RAG 桶没有「过期」或「可信度」的 概念，于是 Agent 反复犯被回滚过的错。这里的记忆是一张 类型化的图（五种节点类型），带指数衰减，并相对当前 pack 检查 grounding 状态——而不是一个扁平的 embedding 存储。 代价：更多活动部件，而且召回是基于衰减、 带 grounding 的，并不是事实来源——它本来就可能旧、 可能错。

在能力面完成之前，先把校准过的状态讲清楚。 Agent 工具里充斥着夸大的说法，而一个说过头的工具，恰恰会 侵蚀它想建立的信任。所以已交付的边界都明写出来： validate-plan 只给 warning（跑干净并不证明 计划成立），remember 是工作区 markdown 交接—— 不是那张类型化记忆图，eval 是脚手架（它不启动 Agent、也不打分；评分卡由人来填），state 是 只读检查。代价：这一页读起来不如含糊其辞 的版本唬人，因为缺口是讲出来的，而不是糊过去的。

用一个回归锁定的基准，而不是「上下文越多一定 越好」。Agent 的上下文到底有没有变好，没有度量 就没有答案，所以 RepoLens 会产出校准过的证据 findings （Agent 可读），并维护一份五个用例、其中一个回归锁定的 eval 契约。代价：这份契约要长期维护， 而且 eval 是脚手架——由人来打分，不是自动证明。

证据

私有快照 origin/main @ 5136bbc——没有可查看的 公开源码、crate 或文档页。支撑这些说法的，是正在运行的 能力面：scan、init、 brief --task、remember --file、 eval list / start、validate-plan --file、 state、compare-summaries、 findings 今天都能通过 CLI 和 MCP 用上，连同 26 个 MCP 工具和那张类型化记忆图。最强的信号，是这个项目 自己的「诚实状态」准则——一张按能力面逐项列出 「已交付 vs 脚手架」的清单，刻意保持校准，好让将来任何 夸大都一眼看得出来——以及一份 dogfood eval 契约（五个用例，其中一个回归锁定），守住上下文质量这条线。

下一步

v0.1 这个里程碑，才是让 RepoLens 作为产品「看得懂」的部分： 更丰富的记忆类型——Hypothesis、 Constraint、FailedAttempt、 FrictionFinding——以及一个明确的 Confidence 枚举（TestValidated / HumanVerified / ScanExtracted / AgentAsserted）。在它们落地到公开 release 或 公开 main 快照之前，这些都还只是规划。

它不是什么

  • 不替你写代码——那是 Agent 的事。
  • 不替代测试或人工 review。
  • 不是通用 RAG 存储——记忆是类型化的、有 grounding、会衰减。
  • 今天还不是多仓库或组织级记忆。
  • 不保证召回出来的记忆一定正确。
  • 私有预发布——今天还不是公开源码项目。
